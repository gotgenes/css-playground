Page Header

The goal is to keep the contents below from causing this main area from expanding beyond the viewport height.

Any scrolling should be constrained to the main contents. This page header should remain at the top of the main container, and the contents footer should remain at the bottom.

The key is to use overflow-y: auto from the contents all the way up to this main container ancestor, to prevent expanding the containing elements from expanding their bounding boxes.
